3. Configuring the anonymization process

In the configuration perspective, data can be imported and transformation rules, privacy models and utility measures can be selected and parameterized.

The perspective is divided into five main areas.

Area 1 shows the current input dataset:
  • The table displays further information on specified attribute metadata.
Area 2 provides a means to specify attribute metadata and view generalization hierarchies:
  • Attribute types and data types can be specified.
  • Generalization hierarchies can be modified.
Area 3 supports the configuration of privacy models:
  • Multiple privacy models can be selected and configured. Further parameters can be specified in dedicated tabs.
Area 4 supports the configuration of utility measures:
  • A single utility measure can be configured and selected as an objective function.
Area 5 provides methods for extracting a research sample:
  • With this concept ARX supports the specification of population tables, by defining the dataset which is to be anonymized as a sample of the dataset that has been loaded.
  • The sample can be selected manually, drawn randomly, or selected by query or by matching with another dataset.
