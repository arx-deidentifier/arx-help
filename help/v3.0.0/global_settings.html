Gobal settings

The settings window is accessible from either the application menu or the application toolbar. The Project tab allows to alter general project properties. The I/O settings tab allows to configure basic properties for the import and export of data. The Transformation tab can be utilized to alter some aspects of the data transformations performed by the tool. When remove outliers is checked, the values of the quasi-identifiers are remove for tuples that are outliers. Otherwise, these values are preserved. To this end, these values are replaced by the suppression string. The maximum size of the search space defines an upper bound for the number of possible transformations, which is defined by the number of quasi-identifiers and the heights of their generalization hierarchies.

The Internals tab allows to adjust some parameters of the optimizations employed by the framework. Changing the number of snapshots enables users to control a space-time-tradeoff provided by the framework. Choosing a value of 1 greatly reduces the main memory requirements of the tool, while leading to potentially significantly increased execution times. Choosing a larger value will reduce execution times, but increase memory consumption. The default value is 200. The Visualization tab allows to control the lattice viewer from the data exploration perspective. The Risk analysis tab allows defining parameter for risk based anonymization. The Solver tab allows to specify parameters for the Newton-Raphson method used for population based risk models.
