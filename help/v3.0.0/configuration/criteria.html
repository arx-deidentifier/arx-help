Privacy criteria (Area 3)

This area allows to specify the desired privacy criteria and the metric used to measure information loss. Criteria can be enabled and disabled by pressing on the plus and minus button respectively.

The k-anonymity, d-presence and the risk based criteria apply to all quasi-identifiers and can therefore always be enabled. This is different for the l-diversity and t-closeness criteria. As they are applied to specific sensitive attributes, they can only be enabled if a sensitive attribute is currently selected (Area 2). The activated criteria a displayed in the table.

k-Anonymity

To further specify the k-anonymity criterion, a slider can be used to define the minimal size of any equivalence class.

l-Diversity

To specify the details of the l-diversity criterion, a slider can be used to define the diversity parameter l. A dropdown list allows selecting the type of l-diversity, i.e., distinct l-diversity, entropy l-diversity or recursive-(c,l)-diversity. If recursive (c,l)-diversity is selected, an additional slider can be used to select an appropriate value for the parameter c.

t-Closeness

To specify the details of the t-Closeness criterion, a dropdown list allows selecting the function that should be used for computing the closeness of data distributions. If "Hierarchical Earth Mover’s Distance" is selected, a generalization hierarchy must be specified for the respective sensitive attribute. A slider enables selecting an appropriate value for the closeness parameter t.

d-Presence

Two sliders can be used for selecting appropriate values for the d-min and d-max parameters. Note that enabling d-presence also requires specifying a research subset to which it is to be applied (Area 5).

Risk based criteria

To configure the risk based criteria a slider can be used to select the risk threshold. If a popultion based risk criteria is selected the underlying population model can also be configured in area 3.
